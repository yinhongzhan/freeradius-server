00066 /*
00067  * EAP_HANDLER is the interface for any EAP-Type.
00068  * Each handler contains information for one specific EAP-Type.
00069  * This way we don't need to change any interfaces in future.
00070  * It is also a list of EAP-request handlers waiting for EAP-response
00071  * eap_id = copy of the eap packet we sent to the
00072  *
00073  * next = pointer to next
00074  * state = state attribute from the reply we sent
00075  * state_len = length of data in the state attribute.
00076  * src_ipaddr = client which sent us the RADIUS request containing
00077  *              this EAP conversation.
00078  * eap_id = copy of EAP id we sent to the client.
00079  * timestamp  = timestamp when this handler was last used.
00080  * identity = Identity, as obtained, from EAP-Identity response.
00081  * request = RADIUS request data structure
00082  * prev_eapds = Previous EAP request, for which eap_ds contains the response.
00083  * eap_ds   = Current EAP response.
00084  * opaque   = EAP-Type holds some data that corresponds to the current
00085  *              EAP-request/response
00086  * free_opaque = To release memory held by opaque,
00087  *              when this handler is timedout & needs to be deleted.
00088  *              It is the responsibility of the specific EAP-TYPE
00089  *              to avoid any memory leaks in opaque
00090  *              Hence this pointer should be provided by the EAP-Type
00091  *              if opaque is not NULL
00092  * status   = finished/onhold/..
00093  */
