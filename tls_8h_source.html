00072 /*
00073  *      A single TLS record may be up to 16384 octets in length, but a
00074  *      TLS message may span multiple TLS records, and a TLS
00075  *      certificate message may in principle be as long as 16MB.
00076  *
00077  *      However, note that in order to protect against reassembly
00078  *      lockup and denial of service attacks, it may be desirable for
00079  *      an implementation to set a maximum size for one such group of
00080  *      TLS messages.
00081  *
00082  *      The TLS Message Length field is four octets, and provides the
00083  *      total length of the TLS message or set of messages that is
00084  *      being fragmented; this simplifies buffer allocation.
00085  */
00086

00250 /*
00251  * From rfc
00252    Flags
00253 
00254       0 1 2 3 4 5 6 7 8
00255       +-+-+-+-+-+-+-+-+
00256       |L M S R R R R R|
00257       +-+-+-+-+-+-+-+-+
00258 
00259       L = Length included
00260       M = More fragments
00261       S = EAP-TLS start
00262       R = Reserved
00263 
00264       The L bit (length included) is set to indicate the presence of the
00265       four octet TLS Message Length field, and MUST be set for the first
00266       fragment of a fragmented TLS message or set of messages. The M bit
00267       (more fragments) is set on all but the last fragment. The S bit
00268       (EAP-TLS start) is set in an EAP-TLS Start message.  This
00269       differentiates the EAP-TLS Start message from a fragment
00270       acknowledgement.
00271 
00272    TLS Message Length
00273 
00274       The TLS Message Length field is four octets, and is present only
00275       if the L bit is set. This field provides the total length of the
00276       TLS message or set of messages that is being fragmented.
00277 
00278    TLS data
00279 
00280       The TLS data consists of the encapsulated TLS packet in TLS record
00281       format.
00282  *
00283  * The data structures present here
00284  * maps only to the typedata in the EAP packet
00285  *
00286  * Based on the L bit flag, first 4 bytes of data indicate the length
00287  */
